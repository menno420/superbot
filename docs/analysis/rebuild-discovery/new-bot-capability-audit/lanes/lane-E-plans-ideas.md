## Ledger conventions

- **Disposition:** `keep` = belongs as a future capability; `improve` = keep but reshape; `merge` = combine into an existing subsystem/plan; `drop` = stale/duplicate/not new-bot-worthy; `redesign` = concept belongs, current form should not; `defer` = useful but not a build commitment yet; `owner-gated` = cannot be promoted without owner/runtime/credential decision.
- **Axis 1 overlap:** `covered-by-Axis-1` means Lanes A–D own shipped-subsystem audit and this row must not be double-counted as a new build. `forward-enhancement` means keep in this Lane E ledger.
- **Manifest sketch language:** sketches use the §2 model from `tools/grammar_spike/spec.py`: `SubsystemManifest`, `CommandSpec`, `PanelSpec`, `SettingSpec`, `StoreSpec`, `EventSpec`, `GatewayListenerSpec` (G-1), `AnnouncementRouteSpec` (G-3), `LeaderboardSpec`, `KnowledgeDomainSpec` (proposed domain family), `ChallengeSessionSpec` / game-session primitive, and `HandlerRef` escape hatches for true domain logic.
- **Outperform target:** `pending Lane F` unless a plan itself names a competitor target. Lane F owns external benchmark discovery; this lane only preserves already-stated targets and obvious comparator slots.
