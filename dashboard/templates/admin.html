Login isn't set up yet

The free multi-user control panel is built and ready, but dormant. To switch it on, set these on the dashboard's Railway service (the bot worker needs CONTROL_API_TOKEN too):

  • DISCORD_OAUTH_CLIENT_ID — the Discord application's client id
  • DISCORD_OAUTH_CLIENT_SECRET — the client secret (set in Railway, never in chat)
  • DISCORD_OAUTH_REDIRECT_URI — https://superbot-dashboard.up.railway.app/auth/callback
  • DASHBOARD_SESSION_SECRET — a long random string for the login cookie
  • CONTROL_API_TOKEN — shared secret, identical on the bot worker + this service

Until then the read-only site works exactly as before. See docs/planning/dashboard-live-editor-plan.md.
