## Shape of the fix

One shared `ABR_CUE_RE` (`utils/btd6/keywords.py`), two consumers:

- **Grounding:** resolver-matched rounds re-fetch via
  `get_round(n, roundset="abr")` on the cue; every `[btd6_round]` line is
  stamped `Round N (ABR)` + an ABR economy note. A silent standard-as-ABR
  mislabel is structurally impossible now; a missing ABR entry says so.
- **Workflow:** `RoundCashPlan` gains `roundset` + `unsupported_modifier`;
  all three executors compute on the plan's roundset, label the economy,
  carry roundset in the evidence id/inputs, and emit an explicit
  "double cash is NOT applied" warning (modifier honesty moved from the
  model's judgment into the deterministic envelope — the BUG-0002/0004
  "deterministic layer owns the label" fix shape again).

Live phrasings now deterministic: r25–r83 ABR = **$113,872.30**
(≈ $119,315.30 projected with the stated $5,443) + the warning; r87-in-ABR
grounds **83,280 RBE / 5 ZOMG (ABR)**. Standard control pinned byte-identical
($19,840).

## Context delta (reflection interview)

1. **Route miss:** none — the bug-book fix sketch (written at capture time,
   previous session) was a perfect entry point; capture-with-sketch is worth
   the extra two minutes every time.
2. **Route excess:** none; the probe + targeted greps carried the whole slice.
3. **Discovered by hand:** the workflow's result dicts are consumed by a
   monkeypatching test (`corrupt()` stub) and an exact-dict inputs assertion —
   adding a field to `normalized_inputs` breaks tests that don't name the
   feature ("round cash" tests, not "roundset"); grep the test file for
   `normalized_inputs ==` before extending evidence shapes.
4. **Decisions made alone:** putting the modifier warning inside the
   deterministic envelope (vs leaving the honesty to the model+contract) —
   in-envelope: it is the established fix shape for this exact class.
5. **Weak point:** the ABR cue is text-global — "ABR" anywhere in the message
   flips every round leg; a question comparing both sets in one sentence
   ("is r87 harder in ABR than standard?") grounds only the ABR side. Rare,
   honest (labeled), and the comparison family is §7.5's scope anyway.
6. **One change that would have helped:** nothing material this time.

## 💡 Session idea (Q-0089)

**Qualifier-coverage audit (the BUG-0010 class, found proactively):**
BUG-0010's shape was "the data layer supports a qualifier; the deterministic
legs ignore it" — and round-set is unlikely to be the only such qualifier.
A small audit (script or invariant test) that enumerates the qualifiers the
dataset/tools expose (round set, difficulty multipliers, mode rules — the
modes data shipped in #655, freeplay scaling, co-op flags if present) and
checks each has BOTH a grounding cue and a workflow/labeling leg — emitting
a `qualifier: supported-by-data / reachable-by-text: yes|no` table — would
surface the rest of this class before live users do, one probe run instead
of one live miss each. Pairs with `btd6_probe.py` (the table is basically a
batch probe) and the bot-self-test-walker idea. (Dedup-checked
`docs/ideas/`: the walker walks commands, the probe replays single texts;
nothing audits qualifier reachability.)
